# 🚀 LearningWebDev

A comprehensive repository capturing my journey from the basics of HTML/CSS to advanced JavaScript and Full-Stack development. This serves as a digital notebook, a collection of practice modules, and a showcase of my technical growth.

---

## 📌 Project Overview
This repository is organized by "Phases," moving from structural design to complex logic. Each directory includes source code, styling experiments, and key takeaways from my technical study sessions.

### 🛠️ Tech Stack
* **Frontend:** HTML5, CSS3 (Flexbox, Grid), JavaScript (ES6+)
* **Backend:** Node.js / Express (Planned)
* **Database:** MongoDB or PostgreSQL (Planned)
* **Tools:** VS Code, Google NotebookLM (for lecture synthesis), Canva (UI/Branding)

---

## 📂 Repository Roadmap

### 🟧 Phase 1: The Foundation (HTML & CSS)
*Status: Completed* ✅
* **Structure:** Semantic HTML5 tags for SEO and accessibility.
* **Styling:** Mastery of the **Box Model** and **Flexbox** layouts.
* **Efficiency:** Using the `font` shorthand property and CSS variables.
* **Responsive:** Building mobile-first designs with Media Queries.

### 🟨 Phase 2: The Brain (JavaScript)
*Status: In Progress* 🏗️
* **Core Logic:** Variables, functions, and logic loops.
* **Interactivity:** DOM Manipulation and Event Listeners.
* **Asynchronous:** Fetching APIs and handling JSON data.
* **Goal:** Building a "Top-Class" multi-page site using only Vanilla JS.

### 🟩 Phase 3: The Engine (Backend & Database)
*Status: Upcoming* 📅
* **Server-Side:** Creating RESTful APIs with Node.js.
* **Data Storage:** Managing user data with Databases.
* **Security:** Learning authentication and secure data handling.

### 🚀 Phase 4: Full-Stack Integration
*Status: Ultimate Goal* 🎯
* Connecting a custom Frontend to a live Backend.
* Deploying a complete web application to a cloud environment.

## 💡 Learning Philosophy
I follow a **"Fundamentals First"** approach. While I use AI tools like **Gemini** and **NotebookLM** to summarize long video lectures into technical cheat sheets for efficiency, I ensure I manually code every layout to build muscle memory. 

> "I believe in mastering the 'Holy Trinity' (HTML/CSS/JS) before moving into modern frameworks like React."
